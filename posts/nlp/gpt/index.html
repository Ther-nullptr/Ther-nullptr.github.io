GPT-1 GPT-2概述

GPT-1（其参数量与GPT-2 small类似）和GPT-2的架构大同小异，主要区别在于预训练方式上。

  • GPT-1的训练数据集为5 GB，GPT-2的训练数据集为40 GB。
  • GPT-1使用无监督pretrain + 有监督finetune，而GPT-2使用无监督pretrain + 无监督finetune，这样使得预训练和finetune的结构一致，同时可以适配不同类型的任务。当问题的输入和输出均为文字时，只需要用特定方法组织不同类型的有标注数据即可代入模型，如对于问答使用“问题+答案+文档”的组织形式，对于翻译使用“英文+法文”形式
  • GPT-2将layer normalization放到每个sub-block之前，并在最后一个Self-attention后再增加一个layer normalization

def forward(self, x):
    B, T, C = x.size() # batch size, sequence length, embedding dimensionality (n_embd)

    # calculate query, key, values for all heads in batch and move head forward to be the batch dim
    q, k ,v  = self.c_attn(x).split(self.n_embd, dim=2)
    k = k.view(B, T, self.n_head, C // self.n_head).transpose(1, 2) # (B, nh, T, hs)
    q = q.view(B, T, self.n_head, C // self.n_head).transpose(1, 2) # (B, nh, T, hs)
    v = v.view(B, T, self.n_head, C // self.n_head).transpose(1, 2) # (B, nh, T, hs)

    # causal self-attention; Self-attend: (B, nh, T, hs) x (B, nh, hs, T) -> (B, nh, T, T)
    att = (q @ k.transpose(-2, -1)) * (1.0 / math.sqrt(k.size(-1)))
    att = att.masked_fill(self.bias[:,:,:T,:T] == 0, float('-inf'))
    att = F.softmax(att, dim=-1)
    att = self.attn_dropout(att)
    y = att @ v # (B, nh, T, T) x (B, nh, T, hs) -> (B, nh, T, hs)
    y = y.transpose(1, 2).contiguous().view(B, T, C) # re-assemble all head outputs side by side

    # output projection
    y = self.resid_dropout(self.c_proj(y))
    return y

注意其中的masked_fill部分。这一部分对attention得到的分数进行上三角式掩码，self.bias的定义如下：

Quantization of GPT

介绍GPT的量化之前，需要先介绍一下BERT的量化。1 一文中指出量化GPT的工作不多（但未说明原因），目前NLP方面的工作主要以量化BERT为主。

Understanding and Overcoming the Challenges of Efficient Transformer Quantization一文2指出了一些目前量化BERT的特点难点：

  • 这些方法都某种形式的QAT，要么不讨论PTQ替代品，要么只把它们作为弱基线。
  • 权重量化造成的损失远小于激活损失。
  • 主要的瓶颈是残余连接中激活张量的不同动态范围之间存在相当大的不匹配，最大的量化损失模块是FFN。
  • 少数异常的词嵌入如[SEP]造成了激活中离群值的出现，导致直接量化整个激活会导致精度降低。

（其实这些问题可能在ViT中也存在，之后找找对应的论文）

对此本文提出的解决方案为：

  • Per-embedding-group PTQ：对每一个嵌入向量的组进行单独量化

  • Mixed precision PTQ：对部分对量化较为敏感的计算进行高位宽处理，如在FFN的输入和输出处使用16 bit，在残差连接处使用16 bit。

需要指出的是，本文的量化均是在微调阶段量化的。

Quadapter一文1指出了GPT量化时的一个问题：大型语言模型的训练数据往往十分宏大，甚至可能只公开一部分数据，然而只在部分数据集上进行量化会导致过拟合。对此该文章引入了adapter机制，允许GPT在部分数据集上进行QAT。值得注意的是，本文是在预训练阶段研究QAT的。
